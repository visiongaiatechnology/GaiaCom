# GaiaCom Code of Conduct

GaiaCom contributors are expected to keep technical discussion respectful,
specific, and security-focused.

## Expected Behavior

- Assume good faith while requiring evidence for security claims.
- Discuss risks and trade-offs directly.
- Keep vulnerability reports private until coordinated disclosure is complete.
- Avoid harassment, personal attacks, or disclosure of private user data.

## Unacceptable Behavior

- Publishing active exploit details before maintainers can respond.
- Submitting real secrets, real user data, or private keys as examples.
- Using project spaces to impersonate maintainers or claim unsupported security guarantees.

Maintainers may remove comments, issues, or contributions that violate these
rules or endanger users.
