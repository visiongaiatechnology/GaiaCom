# GaiaCom Responsible Disclosure Policy - v0.1

We value the contributions of security researchers. This document defines the protocol for reporting vulnerabilities found in the GaiaCom codebase or services.

---

## 1. How to Report a Vulnerability

If you discover a security vulnerability in GaiaCom, please report it privately:

*   **Email Address:** [security@visiongaia.de](mailto:security@visiongaia.de)
*   **Encrypted Communication:** Please encrypt your email using our Form:
  [https://visiongaiatechnology.de](https://visiongaiatechnology.de/#vgt-comlink)
*   **What to Include:**
    *   Detailed description of the vulnerability.
    *   Step-by-step proof-of-concept (PoC) or exploit code.
    *   Affected components (Frontend, Backend, Föderations-Service).

---

## 2. Guidelines & Code of Conduct

We ask researchers to adhere to the following guidelines:

*   **No Public Disclosure:** Do not publish, share, or discuss the vulnerability publicly until we have resolved it and released a patch.
*   **No Destructive Attacks:** Avoid executing denial-of-service (DoS) attacks, brute-force testing, or attempts to access or modify other users' data.
*   **Social Engineering:** Testing must not include social engineering (phishing, vishing) or physical security testing.

---

## 3. Our Commitments

In return for following this policy, we commit to:

*   **Acknowledge Receipt:** Respond to your report within 48 business hours with an initial assessment.
*   **Keep You Informed:** Provide regular updates on our progress as we work on a fix.
*   **Safe Harbor:** We will not pursue legal action against researchers who discover and report vulnerabilities in compliance with these guidelines.
*   **Attribution:** With your permission, we will publicly credit you in our security advisories upon successful mitigation of the reported issue.
